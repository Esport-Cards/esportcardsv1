Mettre Esport Cards en ligne
— tutoriel complet

Par l'équipe Esport Cards · Mai 2026 · 10 min de lecture

De l'application Next.js sur votre machine au site en ligne accessible au monde entier : voici toutes les étapes pour déployer votre plateforme de cartes à collectionner esport.

Vous avez construit une plateforme complète avec Next.js 14, Prisma, NextAuth et une base SQLite. C'est parfait pour le développement local, mais maintenant vous voulez que le monde entier puisse ouvrir des packs, collectionner des cartes et consulter le leaderboard.

Ce tutoriel vous guide étape par étape, du code local vers la production. Aucune connaissance DevOps avancée requise — juste les bonnes commandes au bon moment.

1 Préparer le projet pour la production

SQLite est parfait pour le développement local, mais il ne fonctionne pas en production sur la plupart des hébergeurs modernes. Nous allons passer sur PostgreSQL, la norme industrielle pour les applications Next.js.

1.1 Changer la base de données (SQLite → PostgreSQL)

Modifiez la première ligne de prisma/schema.prisma : 

# Avant :
datasource db {
  provider = "sqlite"
  url      = env("DATABASE_URL")
}

# Après :
datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

1.2 Installer le driver PostgreSQL

npm install pg
npm install --save-dev @types/pg

1.3 Vérifier les scripts package.json

Assurez-vous que vos scripts sont bien configurés :

{
  "scripts": {
    "dev": "next dev",
    "build": "next build",
    "start": "next start",
    "seed": "node prisma/seed.js",
    "prisma:generate": "prisma generate",
    "prisma:push": "prisma db push"
  }
}

💡 Astuce Glass

Créez un fichier .env.example à la racine pour documenter les variables d'environnement sans exposer vos secrets : 

DATABASE_URL="postgresql://user:password@localhost:5432/esportcards"
NEXTAUTH_SECRET="your-secret-key-here"
NEXTAUTH_URL="https://your-domain.com"

2 Choisir l'hébergeur

Pour une application Next.js full-stack (avec API routes, SSR, et base de données), voici les meilleures options en 2026 :

Hébergeur Difficulté Prix /mois Base de données
Vercel ⭐ Recommandé Très facile Gratuit → 20€ À part (Neon / Supabase)
Railway Facile 5€ + usage Inclus
Render Facile Gratuit → 7€ Inclus
VPS (OVH / Scaleway) Complexe 3–5€ À configurer
Nous recommandons Vercel + Neon (PostgreSQL gratuit) — c'est l'option la plus simple et robuste pour Next.js en 2026.

3 Mettre en ligne sur Vercel

Vercel est la plateforme de référence pour Next.js. Le déploiement se fait en quelques clics.

3.1 Pousser le code sur GitHub

# Initialiser git (si ce n'est pas fait)
git init
git add .
git commit -m "Initial commit - Esport Cards MVP"

# Créer un repo sur GitHub puis :
git remote add origin https://github.com/ton-username/esport-cards.git
git push -u origin main

3.2 Créer une base de données PostgreSQL gratuite

Suivez ces étapes :

  1. Allez sur Neon (ou Supabase)
  2. Créez un compte gratuit
  3. Créez un nouveau projet : esport-cards
  4. Copiez la DATABASE_URL (elle ressemble à postgresql://user:password@ep-xxx.neon.tech/neondb)

3.3 Déployer sur Vercel

  1. Allez sur Vercel
  2. Cliquez "Add New Project"
  3. Importez votre repo GitHub esport-cards
  4. Configurez les Environment Variables :
VariableValeur
DATABASE_URLpostgresql://... (celle de Neon)
NEXTAUTH_SECRET(générez une clé : openssl rand -base64 32)
NEXTAUTH_URLhttps://ton-site.vercel.app

5. Cliquez sur "Deploy" — Vercel build votre app et la met en ligne en ~2 minutes.

3.4 Lancer le seed en production

Une fois déployé, vous devez initialiser la base de données avec les vrais joueurs :

# Générer le client Prisma
npx prisma generate

# Pousser le schéma
npx prisma db push

# Lancer le seed (ajoute s1mple, ZywOo, Faker, TenZ...)
npm run seed

💡 Si vous êtes en local, vous pouvez temporairement mettre la DATABASE_URL de prod dans votre .env local pour lancer le seed directement vers la base de production.

4 Alternative : Railway (tout-en-un)

Railway gère à la fois l'app Next.js et la base PostgreSQL dans un seul projet. C'est une excellente alternative à Vercel + Neon.

4.1 Déployer

  1. Allez sur Railway
  2. "New Project" → "Deploy from GitHub repo"
  3. Sélectionnez esport-cards
  4. Railway détecte Next.js automatiquement

4.2 Ajouter PostgreSQL

  1. Dans votre projet Railway → "New Service" → "Database" → "PostgreSQL"
  2. Railway injecte automatiquement DATABASE_URL dans votre app

4.3 Ajouter les variables d'environnement

Dans l'onglet Variables de votre app :

NEXTAUTH_SECRET = (générez une clé sécurisée)
NEXTAUTH_URL    = https://ton-app.railway.app

4.4 Lancer le seed

Dans l'onglet Deploy → "Run Command" :

npx prisma db push && npm run seed

5 Configurer le domaine personnalisé

Donnez une identité premium à votre plateforme avec un nom de domaine personnalisé comme esportcards.fr.

5.1 Acheter un domaine

Recommandés pour la France :

  • OVH (~8€/an)
  • Gandi (~12€/an)
  • Namecheap (~10€/an)

5.2 Lier le domaine sur Vercel

  1. Vercel → Votre projet → Settings → Domains
  2. Ajoutez esportcards.fr et www.esportcards.fr
  3. Vercel vous donne les DNS à configurer chez votre registrar :
Type: A     Nom: @     Valeur: 76.76.21.21
Type: CNAME Nom: www   Valeur: cname.vercel-dns.com

N'oubliez pas de mettre à jour NEXTAUTH_URL dans les variables d'environnement :

NEXTAUTH_URL=https://esportcards.fr

6 Checklist avant de commencer

Vérifiez ces points avant votre premier déploiement :

# 1. Vérifiez que .env.local est dans .gitignore
cat .gitignore | grep ".env"  # doit afficher .env*.local

# 2. Vérifiez que le build passe en local
npm run build

# 3. Vérifiez prisma generate
npx prisma generate

✅ Structure minimale pour la prod

  • prisma/schema.prisma (provider = "postgresql")
  • DATABASE_URL dans les variables d'environnement
  • NEXTAUTH_SECRET défini
  • NEXTAUTH_URL = l'URL de prod
  • npm run build passe sans erreur
  • Pas de .env.local poussés sur GitHub
